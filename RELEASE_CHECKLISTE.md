# Release-Checkliste – psycho-semantischer-guardrail

## Vor dem Release
- [ ] README auf aktuellen Stand bringen
- [ ] Versionsnummer prüfen
- [ ] CITATION.cff validieren
- [ ] .zenodo.json prüfen
- [ ] Lizenzhinweis bestätigen
- [ ] Kerndateien und Beispiele ergänzen

## Beim Release
- [ ] Git-Tag setzen
- [ ] GitHub-Release anlegen
- [ ] Release-Notiz mit Änderungen ergänzen
- [ ] Zenodo-Archivierung prüfen

## Nach dem Release
- [ ] DOI in Portal und Projektdateien ergänzen
- [ ] DOI in CITATION.cff nachtragen
- [ ] DOI in .zenodo.json nachtragen
- [ ] Projektportal aktualisieren
